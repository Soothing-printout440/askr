## 💬 How it works

askr acts like a bridge between your AI client and Q&A tasks. Your client sends a request through MCP. askr handles the request, keeps the flow async, and returns the answer when it is ready.

A simple flow looks like this:

1. Your AI client sends a question.
2. askr receives the request.
3. askr processes the Q&A task.
4. askr sends back the answer.
5. Your client shows the result to you.

This setup helps when you want a clean handoff between the client and the Q&A step.

## 🧩 About MCP support

MCP stands for Model Context Protocol. It helps apps talk to AI clients in a structured way. askr uses MCP so the Q&A flow can fit into tools that support this protocol.

You do not need to learn the full protocol to use the app. You only need the right client and the correct connection settings.
